# インタラクティブな Web セミナーのベストプラクティス {#best-practices-for-interactive-webinars}

小規模なオーディエンスを対象とした仮想イベントを実行する場合、大規模なオーディエンスを対象とした仮想イベントを実行する場合は、予定どおりに進むことが重要です。 イベントの計画と実行に必要な様々な手順が、準備、プロモーション、配信、フォローアップからあるので、多くのような気分になる場合があります。

これに役立つように、成功する魅力的な Web セミナーの計画、設計、作成に役立つベストプラクティスのリストを作成しました。 これらのヒントは、Adobe Connectが提供するインタラクティブウェビナーを最大限に活用するのに役立ちます。

## イベント前の準備 {#pre-event-preparation}

### イベントの作成 {#creating-an-event}

* 一から始めるのを避けるために、サンプルのインタラクティブウェビナー (IW) を作成し、毎回クローンを作成します。

* 後で同じコンテンツを再度配信する場合は、Adobe Connectルームを再利用します。 に移動します。 **ポッドからのコンテンツのクリア** ポッドメニューからチャット履歴を消去し、最後のセッションからポーリング/クイズの結果を消去します。 これにより、部屋の内容がクリアされます。 以前のセッションのすべてのデータは、引き続きMarketo Engageで使用できます。

* バックツーバックセッションのスケジュールを設定しないでください。 ウェビナーを開始する前に、30 分以上の準備時間があることを確認します。 これにより、プレゼンテーションの最後の変更をアップロードし、マイクとカメラを調整し、新しいプレゼンターがインターフェイスを使用して快適に操作できるようになります。

* チームメンバーに役割を割り当てます。 共同ホストおよび発表者として追加します。 主催者と発表者は、参加者の前に部屋に入ることができます。

### イベントのプロモーション {#promoting-your-event}

* ウェビナー用に、Marketoで登録フォームとランディングページを設定します。

* 登録フォームは簡単にしてください。 登録を最大化するには、必要な質問をするだけです。 登録フォーム上の各入力フィールドは、完了を妨げる増分的な障壁として機能します。

* 登録フォームを 1 回デザインし、概要の一貫性を保つために、ウェビナーで再利用するようにします。

* ターゲットオーディエンスに E メールを送信します。 E メールで、主要な価値の提案を簡単に強調する必要があります。 関連トークンを使用してイベントの詳細を保存し、複数の電子メールキャンペーンで再利用できます。 イベントの日時にカレンダートークンを使用し、ウェビナーへの登録後の URL への参加に関する情報を共有します。

* フォローアップ招待メールを設定します。 電子メールの件名行を試します（例：「Don&#39;t Miss Out」、「Seats Are Filling Up」）。

* Web サイト/ブログ、ソーシャルメディア、有料広告、パートナープロモーションを使用して、より広範な閲覧者にリーチし、登録を促進します。

* ウェビナーの 2～3 日前に、登録済みの招待者にリマインダーメールを送信します _および_ イベントの当日に、忘れずに参加できるようにします。

### 部屋を設計する {#designing-a-room}

* 参加者がイベントを開始する前にイベントに入るときに表示される「ロビー」レイアウトを作成します。 このロビーを使用して、マーケティング関連資料などの関連情報を参加者に提供します。 オーディエンスをアイスブレーカーゲームや仲間とのチャットに取り組ませます。 目立つ沈黙を避けるのに役立つ適切なバックグラウンド音楽を再生します。 投票とチャットを使用して、参加者の人口統計や期待など、セッションからの参加者に関するより多くのデータを収集できます。

* レイアウトを使用して、セッションの構造を作成し、モーションを追加します。 例えば、部屋にロビー、議題、スピーカーの紹介、シングルスピーカーのプレゼンテーション、パネルディスカッション、ブレークアウト、Q&amp;A、ラップアップなどのレイアウトを含めることができます。

* エンゲージメントを増やすインタラクションの計画。 調査、Q&amp;A、チャット、クイズは、エンゲージメントを増やす一方で、ファーストパーティデータを生成するのに役立ちます。

* 大きな投票率を期待している場合は、Q&amp;A ポッドに関する質問に回答する責任を持ついくつかの共同ホストを持っていると便利です。 必要に応じて、特定の共同ホストに異なるトピック領域を割り当てることもできます。

* 可能な限り、画面共有ではなく共有ポッドにコンテンツをアップロードします。 共有ポッドで共有および表示できるファイルタイプは、PPT、PPTX、PDF、MP3、MP4、JPG、PNG、POD（カスタムポッドの場合）、有効なコンテンツを含む圧縮パッケージ (ZIP) です。 コンテンツは部屋内で保持され、すぐに使用できます。 これにより、画面の共有に必要なインターネット帯域幅も減少します。

* 100 人を超える参加者を想定している場合は、開いたダイアログ（チャットポッド）からモデレートされたダイアログ（Q&amp;A ポッド）に切り替えることを検討してください。 オーディエンスが大きいと、ホストが監視するのに活発なチャットが困難になる場合があります。

* 最後のレイアウトに「ファイル」ポッドを追加して、参加者にダウンロードしてもらいたいファイル（パンフレット、ホワイトペーパーなど）を共有します。

* Web リンクポッドを使用して、参加者と URL を共有します。 また、参加者にリンクをプッシュして、全員がブラウザーで起動できるようにすることもできます。 これは、セッションの終了に向けて、フィードバック調査、製品体験版の登録、または次のイベントの登録フォームを開始する場合に役立ちます。

* 部屋の基本設定から部屋の背景を設定して、部屋に個性を追加します。 これは、ブランドのロゴを含む壁紙です。 ロゴを隅に揃えるには、整列オプションを使用します。

* セッションを記録する場合は、部屋の基本設定で記録リマインダを有効にします。 記録が開始されていない場合、リマインダーは会議に 5 分間表示されます。

* 「ホストと発表者」領域を有効にし、関連するポッドで設定します。 この領域は、ホストおよびプレゼンターに対してのみ表示され、バックステージでの共同作業に使用できます。 ここに含めるポッドの一部は次のようになります。
   * 参加者のエンゲージメントをリアルタイムで監視するエンゲージメントダッシュボード。 [ここをクリック](https://www.youtube.com/watch?v=gf5fu0JK9Hk) ：エンゲージメントダッシュボードの使用方法に関する短いビデオ。
   * チャットポッド：プレゼンテーションチームメンバー間のプライベートな会話を許可します。
   * メモリマインダーメッセージ、プレゼンター向けの質問、または一般的な質問に対する迅速な回答をコピーして貼り付けることができる標準的な質問の回答を投稿するポッド。

C.配信後 — ニリーシュ

* ウェビナーが配信されたら、記録と共にエンゲージメントデータをMarketoで通常 1～2 時間以内に利用できるようにする必要があります。 録画を検証し、オンデマンドのウェビナーデータを追跡すると同時に、オーディエンスと共有できるように、完全なウェビナービデオ出力が適切に取り込まれたかどうかを確認します。

* インタラクティブなウェビナーのフィルターとトリガーを使用して、参加者のリードをエンゲージメントに基づいてさらに絞り込むための育成キャンペーンを作成し、Thank You Missed You Campaigns で、イベントの詳細、記録リンク、イベント統計、将来のイベントを提供します。 録画を訪問する番組なしのアクティビティはオンデマンド Web セミナーの一部として追跡でき、オンデマンド Web セミナーのフィルターとトリガーを通じて、番組なしのキャンペーンをさらに作成できます。

* 「メンバー」タブの各メンバーのアクティビティログデータを使用して、ウェビナー中に異なる参加者が実行したエンゲージメントアクティビティを確認します。

* 記録をダウンロードし、幅広いオーディエンスリーチを実現するために、様々なランディングページ、マイクロサイト、またはポータルに公開します。

* ダッシュボードを使用して、プログラムのステータス、出席配分、投票応答、およびファイルのダウンロード数とリンクのクリック数に関するウェビナーのパフォーマンスの集計ビューを取得します。 ウェビナーの実施中に、前向きと改善点を特定し、次のウェビナーでレッスンを確実に使用できるようにします。
