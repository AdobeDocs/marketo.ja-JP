# リードを調査 {#investigate-lead}

MQL、プログラムの選定基準、ライフサイクルの進捗を確認しなかった理由を説明します。

調査シナリオの完全なリストを表示するには、右側の矢印をクリックします。

スクリーンショット

| **タイトル** | 説明 |
|----------|------------|
| **MQL調査** | 特定のリードがクオリフィケーションに至らなかった理由、エンゲージメントを停止した理由、行き詰った理由などを詳細に説明します。 |
| **プログラムの選定** | リードがナーチャリングプログラムに適格ではなかった理由の詳細を提供します。 |
| **ライフサイクルの進行** | リードが予想よりも長いステージに留まっている理由を調べるために使用します。 |
| **エンゲージメント分析** | リードが電子メールを開いたりクリックしたりしなくなった場合に使用します。 |
| **キャンペーンジャーニー** | ジャーニー内のリードのフルパスとタッチポイントを表示するために使用します。 |
| **データ品質** | 不正なデータがリードに問題を引き起こしている可能性がある場合に使用します。 |
| **セールスハンドオフ** | 営業担当者がリードを却下し、フォローアップしなかった場合に使用します。 |
| **アトリビューション** | 商談の創出に貢献した要因を把握できます。 |
| **スコア減衰** | リードスコアが予期せず低下した理由を調査します。 |

>[!TIP]
>
>9つのシナリオをすべて簡単に表示するには、上部のプロンプトウィンドウに「`/lead investigation`」と入力します。
